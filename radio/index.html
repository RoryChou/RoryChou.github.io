# Radio

Radio.

# When To Use

  • Used to select a single state in multiple options.
  • The difference between Select is that Radio is visible to user and can facilitate the comparison of choice, which makes there shouldn't be too many of them.

# Examples

# Basic

The simplest use.

<Vue source code> 
<template>
  <section>
    <NebulaRadio>Radio</NebulaRadio>
  </section>
</template>

# Disabled

Radio unavailable.

<Vue source code> 
<template>
  <main>
    <section>
      <NebulaRadio :defaultChecked="false" :disabled="disabled">Disabled</NebulaRadio>
    </section>
    <section>
      <NebulaRadio defaultChecked :disabled="disabled">Disabled</NebulaRadio>
    </section>
    <section>
      <NebulaButton type="primary" @click="toggleDisabled">
        Toggle disabled
      </NebulaButton>
    </section>
  </main>
</template>

<script>
export default {
  data() {
    return {
      disabled: true
    };
  },
  methods: {
    toggleDisabled() {
      this.disabled = !this.disabled;
    }
  }
};
</script>

# Radio Group

A group of radio components.

<Vue source code> 
<template>
  <section>
    <NebulaRadioGroup @change="onChange" v-model="value">
      <NebulaRadio :value="1">A</NebulaRadio>
      <NebulaRadio :value="2">B</NebulaRadio>
      <NebulaRadio :value="3">C</NebulaRadio>
      <NebulaRadio :value="4">D</NebulaRadio>
    </NebulaRadioGroup>
  </section>
</template>

<script>
export default {
  data() {
    return {
      value: 1
    };
  },
  methods: {
    onChange(e) {
      console.log("radio checked", e.target.value);
    }
  }
};
</script>

# Vertical RadioGroup

Vertical RadioGroup, with more radios.

<Vue source code>

      <NebulaRadio :style="radioStyle" :value="3">C</NebulaRadio>
      <NebulaRadio :style="radioStyle" :value="4">D</NebulaRadio>

      console.log('radio checked', e.target.value)

}
</script>

# RadioGroup group - optional

Render radios by configuring options.

<Vue source code> 
<template>
  <main>
    <section>
      <NebulaRadioGroup :options="plainOptions" @change="onChange1" v-model="value1"/>
    </section>
    <section>
      <NebulaRadioGroup :options="options" @change="onChange2" v-model="value2"/>
    </section>
    <section>
      <NebulaRadioGroup :options="optionsWithDisabled" @change="onChange3" v-model="value3"/>
    </section>
  </main>
</template>

<script>
const plainOptions = ["Apple", "Pear", "Orange"];
const options = [
  { label: "Apple", value: "Apple" },
  { label: "Pear", value: "Pear" },
  { label: "Orange", value: "Orange" }
];
const optionsWithDisabled = [
  { label: "Apple", value: "Apple" },
  { label: "Pear", value: "Pear" },
  { label: "Orange", value: "Orange", disabled: false }
];

export default {
  data() {
    return {
      plainOptions,
      options,
      optionsWithDisabled,
      value1: "Apple",
      value2: "Apple",
      value3: "Apple"
    };
  },
  methods: {
    onChange1(e) {
      console.log("radio1 checked", e.target.value);
    },
    onChange2(e) {
      console.log("radio2 checked", e.target.value);
    },
    onChange3(e) {
      console.log("radio3 checked", e.target.value);
    }
  }
};
</script>

# RadioGroup with name

Passing the name property to all input[type="radio"] that are in the same RadioGroup. It is usually used to let the browser see your RadioGroup as a real "group" and keep the default behavior. For example, using left/right keyboard arrow to change your selection that in the same RadioGroup.

<Vue source code> 
<template>
  <section>
    <NebulaRadioGroup name="radiogroup" :defaultValue="1">
      <NebulaRadio :value="1">A</NebulaRadio>
      <NebulaRadio :value="2">B</NebulaRadio>
      <NebulaRadio :value="3">C</NebulaRadio>
      <NebulaRadio :value="4">D</NebulaRadio>
    </NebulaRadioGroup>
  </section>
</template>

# API

# Radio

Property Description Type Default
autoFocus get focus when component mounted boolean false
checked Specifies whether the radio is selected. boolean -
defaultChecked Specifies the initial state: whether or not the radio is selected. boolean false
disabled Disable radio boolean false
value According to value for comparison, to determine whether the selected any -

# RadioGroup

Radio group can wrap a group of Radio。

Property Description Type Default
defaultValue Default selected value any -
disabled Disable all radio buttons boolean false
name The name property of all input[type="radio"] children string -
options set children optional string[] | Array<{ label: string value: string disabled?: boolean }> -
size size for radio button style large | default | small default
value Used for setting the currently selected value. any -
onChange The callback function that is triggered when the state changes. Function(e:Event) -
buttonStyle style type of radio button outline | solid outline
